Data

The Rijksmuseum museum has an API that can be query for information about the collection.

We also used Wikipedia to fill in some additional information about works/artists

Scraping began with the Rijksmuseum’s api. The api was queried for 100 pages of 50 works each. For each work, another api call returned more in-depth information about the work as well as the work’s primary artist. As much information as possible was scraped from the JSON strings returned by these api calls. The information provided by the Rijksmuseum alone was not enough, so we turned to Selenium as well as the Python Wikipedia module to provide us with the missing data fields. For example, Selenium was used to search Google images for the artist’s image as well as for any work whose image was not provided by the Rijksmuseum. Inspired by Google’s “I’m Feeling Lucky” option, we always chose the first image returned by the search. Of course, this method was not perfect, and a lot of times either no image or the wrong image would appear. Overall though, this method returned satisfactory enough results without requiring an unnecessary amount of complexity. The Wikipedia on the other hand worked flawlessly, and allowed us to retrieve the bio “blurbs” for any artist as long as they existed in the database.

The Final step of scraping involved massaging the data and extracting interesting information about subsets of the data such as average dimensions of a medium and common colors in an era. Making meaningful conclusions about common colors presented a challenge, because averaging all the colors in an era would just return some shade of brown, but trying to return the most used colors would probably do something similar since there are many shades which appear in only one or two works. Therefore, our solution was to bucket the colors into approximately twelve buckets. Each bucket was represented by a reference color, such that all the range of visible colors was well represented by the buckets. Each work’s colors were then assigned to buckets based on the Euclidian distance between each reference color and the work color rgb vectors. The shortest distance therefore indicated the reference color that best approximated the true color .
